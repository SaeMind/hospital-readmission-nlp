# Data

## Source

This project uses discharge summaries from **MIMIC-IV** (Medical Information Mart for Intensive Care IV), a publicly available critical care database maintained by the MIT Laboratory for Computational Physiology.

- Homepage: https://physionet.org/content/mimiciv/
- Version used: MIMIC-IV v2.2
- Relevant table: `discharge` (within the `note` module)

## Access Requirements

MIMIC-IV is a credentialed dataset. To obtain access:

1. Complete the **CITI Program** "Data or Specimens Only Research" course
   - URL: https://about.citiprogram.org/
   - Estimated time: 2–3 hours
2. Create a **PhysioNet** account at https://physionet.org/
3. Submit a **credentialing application** on PhysioNet (requires CITI certificate)
4. Once approved, accept the MIMIC-IV Data Use Agreement
5. Download via `wget` or the PhysioNet interface

## Expected Schema

The pipeline expects a CSV with the following columns:

| Column | Type | Description |
|---|---|---|
| `subject_id` | int | De-identified patient identifier |
| `hadm_id` | int | Hospital admission identifier |
| `discharge_summary` | str | Full discharge note text |
| `readmitted_30d` | int | 1 = readmitted within 30 days, 0 = not |

## Data Splits

| Split | File | Records (approx.) |
|---|---|---|
| Train | `train.csv` | 24,000 |
| Validation | `val.csv` | 6,000 |
| Test | `test.csv` | 10,000 |

Splits are generated by `scripts/prepare_data.py` with stratified sampling on the readmission label.

## Note on PHI

MIMIC-IV discharge notes have been de-identified by the source institution. The preprocessing pipeline (`src/text_preprocessor.py`) applies an additional PHI redaction pass as a safeguard. **Never commit raw MIMIC data to this repository.**
